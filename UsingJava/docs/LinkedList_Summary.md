# Linked List (연결 리스트)

## 개념
**Linked List**는 데이터 요소(Node)들이 메모리 상에 연속적으로 존재하지 않고, 각 요소가 다음 요소의 주소(Reference)를 참조하여 연결된 선형 자료구조입니다.

### 구조 (Node)
각 노드는 두 가지 정보를 담고 있습니다:
1.  **Data**: 저장하려는 실제 값.
2.  **Next**: 다음 노드를 가리키는 포인터(참조).

```java
private static class Node<E> {
    E data;
    Node<E> next;
    // ...
}
```

## 장점 vs 단점 (vs Array)

| 특징 | Array / ArrayList | Linked List |
| :--- | :--- | :--- |
| **접근 (Access)** | **빠름 (O(1))** - 인덱스로 바로 접근 | **느림 (O(n))** - 처음부터 순회해야 함 |
| **삽입/삭제 (중간)** | **느림 (O(n))** - 뒤의 요소들을 밀거나 당겨야 함 | **빠름 (O(1))** - 포인터만 변경하면 됨 (단, 위치 탐색은 O(n)) |
| **메모리** | 연속된 공간 필요, 크기 변경 시 복사 비용 | 필요한 만큼 동적 할당, 포인터 저장을 위한 추가 메모리 필요 |

## 구현 핵심 포인트
1.  **Head Node**: 리스트의 시작점을 가리키는 변수가 필요합니다.
2.  **탐색**: 특정 위치의 데이터를 얻으려면 `head`부터 `next`를 타고 이동해야 합니다.
3.  **삭제**: 삭제하려는 노드의 **이전 노드**를 찾아, 그 `next`를 삭제하려는 노드의 `next`로 연결해야 합니다. (`prev.next = current.next`)

---
> [!NOTE]
> 이번 구현은 **단방향(Singly)** 연결 리스트입니다. 실무에서는 앞뒤로 이동이 가능한 **양방향(Doubly)** 연결 리스트(`java.util.LinkedList`)가 더 자주 사용됩니다.
